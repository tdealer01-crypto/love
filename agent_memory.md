# Agent Memory & Core Directives

## Core Objective
To develop a fully Autonomous Agent capable of operating independently on behalf of the user. The agent must be able to perform complex tasks that require interacting with web interfaces, managing local files, and executing tools.

## Current State
- **User Interface:** Telegram messenger.
- **Cognitive Engine:** Google Gemini (gemini-1.5-flash-latest) for natural language understanding and command translation.
- **Security Layer:** A built-in 'LOVE CORE' policy engine that validates every command generated by the LLM against a set of safety rules before execution.
- **Rate Limiting:** A 5-second cooldown mechanism is implemented to prevent API quota errors.
- **Auditing:** All actions are logged in `agent_audit.log`.

## Immediate Goal
**Integrate Web Browsing & Interaction Capabilities.** The next critical step is to equip the agent with tools to control a web browser (likely via Selenium) to enable:
1.  Reading content from web pages.
2.  Analyzing web page structure.
3.  Interacting with web elements (clicking buttons, filling forms).
4.  Uploading and downloading files.

## Session Log
- **Initial Build:** Created a basic Telegram agent that executes shell commands directly.
- **LLM Integration:** Upgraded the agent to use Gemini API to translate natural language (including Thai) into shell commands.
- **Safety Protocol:** Integrated the LOVE CORE policy engine to act as a security checkpoint, rejecting unsafe commands translated by the LLM.
- **Persistent Memory:** This memory file was created to provide long-term context and statefulness to the agent.
